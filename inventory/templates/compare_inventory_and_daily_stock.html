Stock déclaré (gérant) vs stock calculé (système)

Relevé gérant — table daily_stock : chaque jour, pour chaque station, le gérant envoie une ligne (essence + gazoil déclarés). Inventaire système — table inventory : le système enregistre une ou plusieurs lignes selon les opérations (livraisons, ventes pompes, etc.) ; chaque ligne donne le niveau des cuves après l’opération (pas une somme de plusieurs jours). La comparaison aligne le relevé du jour J sur le dernier inventaire système enregistré le même jour J ; s’il n’y en a pas ce jour-là, on affiche le dernier état connu jusqu’à cette date.

Lecture des écarts

  • Jour J : le système a au moins une ligne inventory le même jour que le relevé — les colonnes « Syst. » reprennent le dernier snapshot de ce jour.
  • Cumul : pas d’inventaire ce jour-là ; on affiche le dernier niveau connu jusqu’à cette date (dernière ligne inventory antérieure ou égale).
  • Écart proche de 0 : déclaratif gérant aligné sur l’état système retenu pour cette ligne.
  • Écart important : erreur de saisie, pertes, ou opérations hors application à vérifier.
